A11Y_DOM_01: All image tags require the presence of the alt attribute.

If the image is decorative or redundant to the text on the page, set alt="". If the image is informational, set alt equal to a descriptive alternative for that image. Strings such as "null", "empty", "undefined", and "image" are not appropriate alt attribute values, as they don't communicate anything substantial to the user.

Good Examples:

  • 
<!-- this image has a helpful alt attribute -->
<img id="image_pass1" src="images/img3.jpg" alt="smiling face"/>
                
  • 
<!-- empty alt attribute indicates decorative image -->
<img id="image_pass2" src="images/img4.jpg" alt=""/>

Bad Examples

  • 
<!-- this image has no alt attribute -->
<img id="image_fail1" src="images/img1.jpg"/>
                  
  • 
<!-- this image has an alt attribute, but it isn't helpful -->
<img id="image_fail2" src="images/img2.jpg" alt="image"/>

A11Y_DOM_02: There must be a one-to-one relationship between labels and inputs.

Every input, textarea, and select must have an associated, non-empty label element, which helps users of assistive technology know which input field is which. The label can be associated in one of two ways:

  • explicit: the label has a for attribute whose value is the input element's id
  • implicit: the label element wraps around the input element
The label tag shouldn't be used without a form field.

Good Examples:

  • 
<!-- input field with non-empty implicit label -->
<label>
          First Name
<input id="input_pass1" type="text"/>
</label>
                  
  • 
<!-- input field with non-empty explicit label -->
<label for="input_pass2">
          Last Name
</label>
<input id="input_pass2" type="text"/>

Bad Examples:

  • 
<!-- input field with no label -->
<input id="input_fail1" type="text"/>
          
  • 
<!-- input field with empty implicit label -->
<label>  
  <input id="input_fail2" type="text"/>
</label>
          
  • 
<!-- input field with empty explicit label -->
<input id="input_fail3" type="text"/>
<label for="input_fail3"></label>
          
  • 
<-- select with unassociated label -->
<label id="unassociated_label1">Do you like pizza?</label>
<select id="input_fail4">
  <option>Yes</option>
  <option>No</option>
  <option>Maybe</option>
</select>
          
  • 
<-- textarea with unassociated label -->
<label id="unassociated_label2">Comment</label>
<textarea id="input_fail5"></textarea>

A11Y_DOM_03: Buttons must have non-empty text labels.

Every button in the UI should have a non-empty string of text in them. If the button does not have any visual text (ex. it has an icon or image inside it), it still must have some hidden text for assistive technology. This can be achieved by using an alt attribute for the button's image or by including a text span that is visually hidden but still accessible to screen readers.

Note that "&nbsp;" does not count as non-empty text.

Good Examples:

  • 
<button id="button_pass1">
  <img src="images/favorite.png" alt="Favorite"/>
</button>
                  
  • 
<!-- this button has hidden text -->
<button id="button_pass2">
  <img src="images/favorite.png" alt=""/>
  <span class="hiddenText">Favorite</span>
</button>
                  
  • 
<-- using aria-label -->
<button id="button_pass3" aria-label="Favorite">
  <img src="images/favorite.png" alt=""/>
</button>

Bad Examples

  • 
<-- button has nothing in it -->
<button id="button_fail1"></button>
                  
  • 
<-- button has an image in it but no text -->
<button id="button_fail2">
  <img src="images/favorite.png" id="img_button_fail"/>
</button>
                  
  • 
<-- button has an image with an empty alt -->
<button id="button_fail3">
  <img src="images/favorite.png" alt=" "/>
</button>
                  
  • 
<-- button has an empty aria-label and its image has an empty alt -->
<button id="button_fail4" aria-label=" ">
  <img src="images/favorite.png" alt=" "/>
</button>

A11Y_DOM_11: Non-interactive DOM elements should not have onclick events.

Onclick events should only be attached to the following HTML elements: a, button, input, canvas.

Clickable divs, spans, and other non-actionable elements should not be made clickable, as it is impossible for users of various assistive technology to click them. You can wrap an anchor (<a>) tag around an element and assign your element and assign the onclick event to the anchor. Another workaround is to add a child anchor tag and attach the same onclick event to the anchor if you want to keep onclick event on your elements.

Click Counter: 0

Good Examples:

  • 
<button id="buttonOnClick" onclick="clicked()">Click this button</button>
  • anchor onclick
    
<div>
  <a href="javascript:void()" onclick="clicked()">anchor onclick</a>
</div>

Bad Examples:

  • Click this div
    
<div id="divOnClick" onclick="clicked()">Click this div</div>
  • Click this span 
    
<span id="spanOnClick" onClick="clicked()">Click this span</span>
  • Click anywhere in this fieldset Don't be shy 
    
<fieldset id="fieldsetOnClick" onClick="clicked()">
  <legend>Click anywhere in this fieldset</legend>
  Don't be shy
</fieldset>
